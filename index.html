GET /api/v2/login¶
해당 API
  1. 토큰 따기
  1. GET /api/v2/login
  1. 로그인하기
  1. POST /api/v2/login
로그인 로직
  1. GET /api/v2/login에 접속
  2. 로그인 Salt를 받는다.
  3. 클라이언트에서는 sha256(salt + sha256(password) + salt) 값을 만들어 서버에 전송한다.
  4. Password 테이블 값과 비교하여 일치하면 session 값들을 고쳐준다.
GET /api/login or /api/v2/login
Parameters
Nothing
Response
200
1
2
3
{
  "identifier": "a3Bd1g", // Password Salt
}

Parameters
  1. “email”: 유저 email 주소 (ciceron_lib.get_user_id를 통하여 email에서 user_id를 추출할 수 있다.)
  2. “password”: 3번 참조
Response
200 로그인 성공
1
2
3
4
5
{
  "logged_in": true, // 로그인 상태
  "useremail": "blahblah@ciceron.me", // 로그인한 유저 메일

GET /api/v2/user/payment/postprocess¶

결제 진행 이후 내부에서 결제 완료 처리하는 API.

Paypal, Alipay같은 경우 결제 완료 후 Redirect할 주소를 넣어 결제 이후의 내부 쇼핑몰과 연결고리를 만들게 된다.

Frontend에서 직접 call할 일은 없다.

POST /api/v2/user/payment/start¶

결제 진행 API

Parameters
  1. “payment_platform”: 지불 플랫폼 (“alipay”, “paypal”, “iamport”, “point”)
  2. “product”: 구매한 내부 플랫폼 이름 (“l10n”, “pretranslation”, “groupRequest”, “f2finterpreter”)
  3. “request_id”: 각 플랫폼에서의 의뢰 번호
  4. “amount”: 지불 금액 (단위: USD)
  5. “promo_type”: 쿠폰 타입 (“indiv”: 개인적용, “common”: 마구 뿌린 프로모션)
  6. “point_for_use”: 결제시 사용할 포인트. 사용한 만큼 결제금에서 차감
  7. “card_number”: 카드번호 (iamport에서만 필요,”xxxx-xxxx-xxxx-xxxx” or “xxxx-xxxx-xxxx-xxx”[AMEX] )
  8. “expiry”: 유효기간 (iamport에서만 필요, “YYYY-MM”)
  9. “birth”: [개인] 생년월일 “YYMMDD”, [사업자] 사업자등록번호 “xxxxxxxxxx” (iamport 전용)
  10. “pwd_2digit”: 비밀번호 앞 2자리 (iamport 전용)
  11. “buyer_name”: 구매자 이름 (iamport 전용, OPTIONAL)
  12. “buyer_email”: 구매자 이메일 (iamport 전용, OPTIONAL)
Response
200
1
2
3
{
  "link": "http://payment.paypalalipay.com/paymemnt/blahblah" // 리다이렉팅 할 주소
}

410: 포인트 보유량이 사용할 포인트보다 적음 411: 유효하지 않은 쿠폰 번호 412: 결제준비 실패
